Step 1: Dataset Summary & Exploration¶

The pickled data is a dictionary with 4 key/value pairs:

  • 'features' is a 4D array containing raw pixel data of the traffic sign images, (num examples, width, height, channels).
  • 'labels' is a 1D array containing the label/class id of the traffic sign. The file signnames.csv contains id -> name mappings for each id.
  • 'sizes' is a list containing tuples, (width, height) representing the original width and height the image.
  • 'coords' is a list containing tuples, (x1, y1, x2, y2) representing coordinates of a bounding box around the sign in the image. These coordinates assume the original image. The pickled data contains resized versions (32 by 32) of these images.

Below, we provide a basic summary of our dataset.

We see that there are 34,799 training examples and 12,630 testing samples. Each image is 32 pixels by 32 pixels. There are 43 classes for the images, i.e. there are 43 different traffic signs in the dataset.

Step 2: Design and Test a Model Architecture¶

We design and implement a deep learning model that learns to recognize traffic signs. The model is trained and tested on the German Traffic Sign Dataset.

We use the classic Lenet-5 architecture. The key modifications we made in order to increase the validation accuracy to over 0.930 was the inclusion of dropout after each of the two fully connected layers. Our validation accuracy is found to be over 0.930 and preprocessing the data. For preprocessing the data, we convert the images to grayscale and normalize them.

Pre-process the Data Set (normalization, grayscale, etc.)¶

For the preprocessing of the data, we choose to normalize the images and convert to grayscale. Normalization is used to ensure all the images are in a similar pixel range. This also helps make photos that might have high glare or other problems easier to process and understand. We convert to grayscale because it simplifies the images significantly and reduces processing time.

import cv2 
from numpy import newaxis

# function which converts an image to grayscale
def grayscale(image):
    image = cv2.cvtColor(image, cv2.COLOR_BGR2GRAY)
    return image

def normalize(image):
    minimum = 0.1
    output = image / 255 * 0.8 + minimum
    return output

def preprocess(data):
    processed = []
    for image in data:
        processed.append(grayscale(image))
    return np.array(processed)

Model Architecture¶

We choose a batch size of 128 with 30 epochs. The batch size is chosen not to be larger, so that the model does not lose the ability to generalize. A smaller batch size is not chosen lest there are too few examples. We settled on 128, though a similar number would work similarly well. We choose 30 epochs, because when training the model for less epochs, we found that the model was still improving signifantly even when the model was done. After 30 epochs, it seems the model reaches a validation accuracy which it would hover around even if additional epochs were added.

We began by using the classic Lenet-5 architecture. This led to a validation accuracy of around 0.88. We then preprocessed the images and used the same architecture, only adjusting slightly to account for the new input pixel size. This helped increased the validation accuracy, but there seemed to be overfitting, comparing the validation accruacy to the test set accuracy. The next adjustment was to add two dropout layers after each of the fully connected layers to prevent overfitting. This is how we reached the model that is seen above.

### We use the Lenet-5 architecture but add two dropouts after the two fully connected layers.
from tensorflow.contrib.layers import flatten

def LeNet(x):    
    # Arguments used for tf.truncated_normal, randomly defines variables for the weights and biases for each layer
    mu = 0
    sigma = 0.1
    
    # Layer 1: Convolutional. Input = 32x32x1. Output = 28x28x6.
    conv1_W = tf.Variable(tf.truncated_normal(shape=(5, 5, 1, 6), mean = mu, stddev = sigma))
    conv1_b = tf.Variable(tf.zeros(6))
    conv1   = tf.nn.conv2d(x, conv1_W, strides=[1, 1, 1, 1], padding='VALID') + conv1_b
    
    # Activation.
    conv1 = tf.nn.relu(conv1)

    # Pooling. Input = 28x28x6. Output = 14x14x6.
    conv1 = tf.nn.max_pool(conv1, ksize=[1, 2, 2, 1], strides=[1, 2, 2, 1], padding='VALID')

    # Layer 2: Convolutional. Output = 10x10x16.
    conv2_W = tf.Variable(tf.truncated_normal(shape=(5, 5, 6, 16), mean = mu, stddev = sigma))
    conv2_b = tf.Variable(tf.zeros(16))
    conv2   = tf.nn.conv2d(conv1, conv2_W, strides=[1, 1, 1, 1], padding='VALID') + conv2_b
    
    # Activation.
    conv2 = tf.nn.relu(conv2)

    # Pooling. Input = 10x10x16. Output = 5x5x16.
    conv2 = tf.nn.max_pool(conv2, ksize=[1, 2, 2, 1], strides=[1, 2, 2, 1], padding='VALID')

    # Flatten. Input = 5x5x16. Output = 400.
    fc0   = flatten(conv2)
    
    # Layer 3: Fully Connected. Input = 400. Output = 120.
    fc1_W = tf.Variable(tf.truncated_normal(shape=(400, 120), mean = mu, stddev = sigma))
    fc1_b = tf.Variable(tf.zeros(120))
    fc1   = tf.matmul(fc0, fc1_W) + fc1_b

    # Activation.
    fc1    = tf.nn.relu(fc1)
    
    # Dropout.
    fc1 = tf.nn.dropout(fc1, keep)

    # Layer 4: Fully Connected. Input = 120. Output = 84.
    fc2_W  = tf.Variable(tf.truncated_normal(shape=(120, 84), mean = mu, stddev = sigma))
    fc2_b  = tf.Variable(tf.zeros(84))
    fc2    = tf.matmul(fc1, fc2_W) + fc2_b
    
    # Activation.
    fc2    = tf.nn.relu(fc2)

    # Dropout.
    fc2 = tf.nn.dropout(fc2, keep)
    
    # Layer 5: Fully Connected. Input = 84. Output = 10.
    fc3_W  = tf.Variable(tf.truncated_normal(shape=(84, 43), mean = mu, stddev = sigma))
    fc3_b  = tf.Variable(tf.zeros(43))
    logits = tf.matmul(fc2, fc3_W) + fc3_b
    
    return logits

Train, Validate and Test the Model¶

We apply TensorFlow's cross entropy function to the Logits, and also use the Adam Optimizer with a learning rate of 0.005. For the learning rate, we attempted training the model with a rate of 0.001 and a rate of 0.01, but found that 0.005 provided the best results.

loss_operation = tf.reduce_mean(cross_entropy)
optimizer = tf.train.AdamOptimizer(learning_rate = rate)
training_operation = optimizer.minimize(loss_operation)

### This is the function to test our validation accuracy.
correct_prediction = tf.equal(tf.argmax(logits, 1), tf.argmax(one_hot_y, 1))
accuracy_operation = tf.reduce_mean(tf.cast(correct_prediction, tf.float32))
saver = tf.train.Saver()

def evaluate(X_data, y_data):
    example_count = len(X_data)
    total_accuracy = 0
    sess = tf.get_default_session()
    for offset in range(0, example_count, BATCH_SIZE):
        batch_x, batch_y = X_data[offset:offset+BATCH_SIZE], y_data[offset:offset+BATCH_SIZE]
        accuracy = sess.run(accuracy_operation, feed_dict={x: batch_x, y: batch_y, keep: 1.0})
        total_accuracy += (accuracy * len(batch_x))
    return total_accuracy / example_count

A test set is used to assess how well the model is performing. A low accuracy on the training and test sets imply underfitting. A high accuracy on the training set but low accuracy on the test set implies overfitting. We find a test set accuracy of 0.9087 which implies slight overfitting.
